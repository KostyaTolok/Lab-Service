Конфигурация находится в файлах config.xml и config.json, файл валидации xml лежит в configValidation.xsd
При запуске программы создается объект OptionsManager, который создает xml или json Parser, по пути переданному ему через конструктор.
Parser создает объект настроек, считывает xml или json документ, валидирует его, устанавливает считанные свойства объекту настроек через метод
SetProperties и возвращает этот объект который присваивается одной из моделей настроек, которые находятся в классе Options. Далее поля 
моделей используются в работе службы. У полей есть атрибуты, которые устанавливают ограничения на поля и задают стандартные значения,
которые устанавливаются в случае возникновения исключительной ситуации во время работы парсера. Также при возникновении исключительной
ситуации Logger записывает исключение в sourcelog.
