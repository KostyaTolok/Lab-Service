Конфигурация находится в файлах config.xml и config.json,схема валидации xml находится в configValidation.xsd
При запуске программы в классе FileWatcher создается объект OptionsManager, который создает xml или json Parser, по пути переданному ему через конструктор.
Parser - это абстрактный класс который объявляет метод GetProperty. XmlParser или JsonParser переопределяет этот метод и получает значение свойства в формате строки.
Поиск свойства в файле конфигурации осуществляется по атрибуту Name из моделей Options.
Parser создает объект настроек, считывает xml или json документ, валидирует его, устанавливает считанные свойства объекту настроек через метод
SetProperties и возвращает этот объект который присваивается одной из моделей настроек, которые находятся в классе Options. Далее свойства
моделей используются в работе службы в классе Service. У полей есть атрибуты, которые устанавливают ограничения на поля и задают стандартные значения,
которые устанавливаются в случае возникновения исключительной ситуации во время работы парсера. Присутсвуют атрибуты CompressionLevel и Directory, которые проверяют уровень
сжатия и путь к папке. Также при возникновении исключительной ситуации Logger записывает исключение в sourcelog.
